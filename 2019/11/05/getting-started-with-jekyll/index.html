Getting Started With Jekyll

Nov 5, 2019 • jekyll, site

So I am finally beginning to feel comfortable with the Jekyll workflow. Here I want to document all the pieces of the setup process in the case I migrate machines in the future.

I will be assuming you are on Mac OS X, though you can probably follow along on Linux by substituting apt-get in place of brew.

Install Ruby, Jekyll, and Bundler

Following the procedures here, we install Ruby

brew install ruby
echo 'export PATH="/usr/local/opt/ruby/bin:$PATH"' >> ~/.zshrc

and then install Jekyll.

gem install --user-install bundler jekyll
echo "export PATH=$HOME/.gem/ruby/2.6.0/bin:$PATH" >> ~/.zshrc

Then we make a new blog

jekyll new myblog
cd myblog

Install KaTeX, Scholar, Themes, and Other Plugins

Add these lines into your Gemfile.

group :jekyll_plugins do
  gem "jekyll-feed"
  gem "jekyll-katex"
  gem "jekyll-scholar"
  gem "jekyll-whiteglass"
end

then run

bundle install

Finally, add the KaTeX CSS and Fonts to your theme. We’re going to overwrite the theme’s default headers by making a _includes folder and adding custom code to our header.html. So do the following:

mkdir _includes
cd _includes
vim header.html

and then paste in

<header class="site-header">

  <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/katex@0.11.1/dist/katex.min.css" integrity="sha384-zB1R0rpPzHqg7Kpt0Aljp8JPLqbXI3bhnPWROx27a9N0Ll6ZP/+DiW/UqRcLbRjq" crossorigin="anonymous">
  <div class="wrapper">

    <a class="site-title" href="/">Dmitry Shemetov</a>

    <nav class="site-nav">
      
        
        <a class="page-link" href="/about/">About</a>
      
        
        <a class="page-link" href="/blog/">Blog</a>
      
        
        <a class="page-link" href="https://github.com/dshemetov/">GitHub</a>
      
    </nav>

  </div>

</header>

(If you are using a different theme, find your theme’s default header file, copy it to _includes, and add the katex.min.css part.)

Finally, to use the scholar plugin create a _bibliography folder and put a references.bib in there

mkdir _bibliography
